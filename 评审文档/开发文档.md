## 1. 项目是什么

**指北 TrueNorth** 是一个单文件 AI 内容导航站（`TrueNorth.html`），把分散的 AI 文章、案例、工具、概念整理成一个互相关联的知识网络，核心目标是带用户走完一条「**看懂 → 跑通**」的路径。

定位升级路线（已落地）：
- 从「AI 内容资料库」→「AI 工作导航站」→「Skill-first 的可执行能力层」。

六大内容层 + 一个能力层：
- **AI 信号**：发现近期变化（模型 / 工具 / 论文 / 行业）。
- **关键洞察**：跨来源提炼判断（数据 / 变化 / 工具 / 框架四类）。
- **实战案例**：企业 / 团队 / 个人怎么真用 AI。
- **方案库（Toolkit）**：项目 / 工具 / 平台 / Agent / Skill 集合。
- **AI 词典**：72 个概念，7 站学习路线 + 故事 / 卡片模式。
- **来源图谱**：来源机构 / 平台的可信度与溯源。
- **论文写作工作台（#paper）**：面向论文 / 研究场景的聚合视图——把库内对写论文有用的 Skill 与工具按「选题 → 检索综述 → 读文献 → 实证 → 修改答辩」分阶段排好（数据见 `PAPER_MAP`，`renderPaper()` 渲染，卡片点击直达对应 Skill / 工具详情）。
- **公考备考 SOP（#gongkao · 测试模块）**：本站唯一**非 skill-first、原创整理**的模块。覆盖国考 / 省考 / 选调（含紧急选调）/ 事业编等（殊途同归、差异在报考条件与命题侧重），按 7 阶段排出备考主线（数据见 `GK_TYPES` / `GK_SOP`，`renderGongkao()` 渲染）；显著标注「🧪 测试阶段」、不构成报考建议，真实契合处借用现有 Skill（chip → `openSkillDetail`，点击直接打开 Skill 详情抽屉，与论文工作台一致）。
- **Skill 库（SKILL_LIBRARY）**：Skill-first 的「可执行能力层」——一个个可理解、可复制、可使用的 Skill。

## 2. 技术栈与约束

- **单文件**：`TrueNorth.html`，内联 CSS + JS，无构建、无依赖、无后端、无数据库。
- **离线自包含**：品牌 logo 以 base64 内联在 `LOGOS` 对象；不做前端实时网络请求（爬取一律离线脚本完成）。
- **两个 `<script>` 块**：①数据层（DATA）②渲染与交互层（RENDER/LOGIC）。务必保持**单个 `<style>`、单数据块、单逻辑块**，不要出现嵌套 / 未闭合标签。
- **校验方式**：抽出两个 `<script>` 用 `node --check` 跑语法；本项目每次改动后都会跑。

### 4.1 方案库的「合并层」设计（关键）

为了**不改动庞大的 `TOOLKIT` 原数组**，所有新字段放在按 id 合并的旁挂对象里，`tkData(id)` 在渲染时合并：

- `TK_EXTRA[id]`：task / level / time / status / source / output / bestFor / notFor / prepare / quickStart / copyable / notes / relatedGlossary / **guides（平台用法子条目）**
- `TK_TYPE[id]`：project / tool / skill_collection / single_skill / agent / platform / workflow / template
- `TK_TASK_ADD[id]`：在不改原 task 的前提下追加任务标签（如 agent / rag）
- `TK_QUALITY[id]`：sourceReliability / runnable / verifiedByTrueNorth / riskLevel / note
- `TK_SKILLS[id]`：Skill 集合的子 Skill 目录（样本）
- `TK_STATS[id]`：来源真实总数（skillCount，可远大于展示样本数）
- `TK_DEPLOY[id]`：跑通路径（deployable / type / aiTools / requirements / manualSteps / codexPrompt / kiroPrompt / claudeCodePrompt / acceptanceCriteria / riskNotes）

`tkData()` 同时做**旧字段兼容**：`url→sourceUrl`、`desc→summary`、`scene→scenario`、`slides→interpretation.slides`、`quickStart→tutorial.available`。

### 4.2 SKILL_LIBRARY 字段（Skill-first）

```
{ id, name, desc, task[], scenario, difficulty, input, output, install, prompt,
  tags[], whyTrueNorth, sourceRefs[{title,url,sourceType,sourceName,extractedFrom}],
  reviewStatus:'pending|reviewed|rejected', riskLevel, note,
  relatedTool, relatedGlossary[] }
```
- **Skill 是最小入库单位**；`sourceUrl` 只是来源，不等于项目。普通文章只作为 `sourceRef`；`parentProjectId`/`relatedTool` 仅在明确属于某集合/平台时使用。
- 前端只展示 `reviewed`（`SHOW_PENDING_SKILLS=false`）；`rejected` 永不展示；缺失 `reviewStatus` 的旧数据按 `reviewed` 兼容。

### 4.3 跨模块关系层

- `GLOSSARY_REL[en]`：词条 → tools / cases / signals / skills
- `CASE_REL[id]`：案例 → tools / glossary / skills
- `SIGNAL_REL[id]`：信号 → skills（数据较稀，缺失则不显示）
- 平台 → 相关 Skill：`SKILL_LIBRARY` 中 `relatedTool===平台id` 自动出现在该平台详情。

## 5. 渲染与交互（第二个 `<script>`）

- 列表渲染：`renderCards`(内容库) / `renderFeed`(案例) / `renderSignals`(信号) / `renderTk`(方案库) / `renderSkillLib`(Skill 库) / `renderGloss`(词典 7 站) / `renderDigest`(洞察) / `renderMap`(来源图谱)。
- **Deck 沉浸式解读**：`openDeck/closeDeck/deckGoTo`，翻页式，主题色渲染。
- **方案库详情抽屉**：`openTkDetail(id, focus)`——适合谁/不适合、产物、准备、3 分钟快速开始、可复制命令/Prompt、**跑通路径**(手动/Codex/Kiro/Claude Code 分 Tab)、**Skill 目录**、**平台用法子条目（带指北解读）**、相关案例/概念/信号、来源信息、下一步你可以。
- **Skill 详情抽屉**：`openSkillDetail(id)`——复用同一浮层，展示场景/输入/输出、安装、Prompt、whyTrueNorth、多 sourceRefs（含 extractedFrom）、相关工具/概念。
- **方案库视图切换**：`tkView` = projects | skills，`setTkView()`；任务入口 `tkTask`（10 个）与搜索 `tkSearch` 对两个视图都生效。
- **跨模块跳转**：`goSkill(skillId)`（跳 Skill 库并高亮）；`gotoTerm(en)`（跳词典）；洞察 `tk` 字段可指向方案库详情。
- **全站搜索数据方法**：`searchAll(query)`，按 signal/insight/case/toolkit/skill/glossary/source 分组（含 SKILL_LIBRARY 的 sourceRefs 字段）；**已配套独立搜索 UI**（导航「⌕ 搜索」+ Ctrl/⌘ K 浮层，复用此方法，结果分组可点击直达）。
- **折叠**：除导航/核心区（quickstart/contentpath/signals/insights/map/about）外，模块默认折叠、点标题展开。

## 7. 内容生产流水线（离线，不在前端跑）

> **重要：当前 SKILL_LIBRARY 的 10 条为人工精选入库；以下流水线是后续批量扩展时使用的离线辅助流程，不是前端运行逻辑，也不是当前唯一入库方式。**
>
> - **当前主要方式**：人工精选已有 Skill → 补 whyTrueNorth → 补 sourceRefs → 贴进 SKILL_LIBRARY → 前端验证。
> - **后续扩展方式**：脚本辅助提取候选 → 人工复核（reviewed/rejected）→ 合并入库。

```
jobs.json → extract-skills.py（默认 dry-run）→ --fetch → data/skill_candidates.json（全 pending）
→ 人工复核（reviewed/rejected，补 whyTrueNorth）→ merge-reviewed-skills.py
→ data/skill_library_reviewed.js（const SKILL_LIBRARY_REVIEWED=[...]）→ 人工贴回前端 SKILL_LIBRARY
```
安全规则：默认不联网；`--fetch` 每请求≥3 秒、遇 401/403/登录页跳过、不递归、不绕权限；候选一律 `pending`，脚本永不自动 `reviewed`、永不直接改 `TrueNorth.html`；医疗/金融/法律自动 `riskLevel:high` + 免责声明。详见 `内容流水线/skill-extraction.md`。

当前阶段是**小批量链路验证**，不是大规模收录。

建议阈值（任一超过即评估拆分）：

| 数据 | 阈值 |
| --- | --- |
| SKILL_LIBRARY | > 30 |
| TOOLKIT | > 60 |
| CURATED | > 20 |
| SIGNALS | > 50 |
| GLOSSARY | > 100 |

拆分顺序建议：先拆 `SKILL_LIBRARY` 与 `TOOLKIT`（增长最快），再拆其余；保持「数据层 / 渲染层」清晰分离即可，不必引入框架。
