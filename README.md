## 为什么做这个

AI 内容太多，能落地的太少。多数人卡在同一条沟里：**看了很多，仍然不知道明天上班能改哪一步。**

指北的答案是把分散信息压成一条可以走的主线：

```
看懂趋势 → 找到方法 → 看到真实案例 → 拿到工具 / Skill → 今天就跑通一个
```

站内一切设计都服务这条主线：信号有出处、案例可溯源、方案带「指北解读」、Skill 给出装法与提示词——**不裸挂链接，不搬运焦虑**。

## 30 秒上手

1. 打开 [线上站](https://true-north-ev.netlify.app)，或克隆本仓库后**直接双击 `TrueNorth.html`**（任意现代浏览器，无需服务器与联网，logo 已 base64 内联）；
2. 按 `Ctrl / ⌘ K` 全站搜索，或从首页「开始」区选一条路线；
3. 赶时间就走最短路径：**Skill 库 → 挑一条「新手友好」 → 复制提示词 / 安装命令 → 去你的 AI 里跑**。

## Skill-first 是什么意思

「Skill」是这个站的一等公民：**比文章小、比工具轻、拿来即用的一段可复用能力**（一条提示词、一个安装命令、一套追问节奏）。

入库标准（详见 `评审文档/`）：

- **不原创、必接地**——每条 Skill 接地真实 GitHub 仓库或权威来源，可溯源（公考 SOP 为显式破例，已标注测试）；
- **人工精选、全量复核**——45 条全部 `reviewed`，聚合库负责「多」，这里负责「能跑」；
- **不裸挂链接**——平台与工具必须带「指北解读」：它是什么、适合谁、不适合谁、我们的判断；
- **诚实边界**——风险等级、适用人群、不承诺效果。

## 工程形态：为什么是一个 HTML 文件

```
TrueNorth.html  (~713 KB)
├── <style>    × 1     全站样式
├── <script>   × 1     数据层（内容即数据：SIGNALS / TOOLKIT / SKILL_LIBRARY / GLOSSARY…）
└── <script>   × 1     渲染交互层（搜索 / 图谱 / Deck / 关系网络）
```

- **可整文件转发**：发一个文件 = 发整个站，收到就能开，不依赖我的服务器活着；
- **离线优先**：无构建、无后端、无外部依赖，飞机上也能用；
- **数据与渲染分离**：内容以结构化数组维护，改内容不碰渲染层。

代价与取舍：单文件体积随内容增长、协作编辑门槛高——对「个人维护 + 整文件分发」的定位，这是划算的交换。Skill 已过 30 条阈值，评估后**暂不拆分**，待真有维护痛点再以可逆方式拆。

## 质量与验证

- **系统级验收**：MVP v1 已 Review Freeze，无未解决 P0（`评审文档/Review_Report.md`）；
- **每次改动**：抽出两个 `<script>` 过 `node --check`，再用 jsdom 真实加载复验，**0 error / 0 warning 才算过**；
- **外部走查**：浏览器走查报告与外部 reviewer 反馈均在 `评审文档/` 留档；
- **反馈闭环**：站内反馈表单（Netlify Forms）直达作者。

## 常见问题

**为什么不做成多页站 / 加个后端？**
分发形态优先。单文件让「分享整个站」和「转发一个文件」是同一件事，且永远离线可用。

**内容多久更新？**
不追热点节奏，按「有信息增量才入库」更新；每次扩容都走同一套复核与验证流程。

**可以贡献吗？**
个人项目，暂不接 PR；欢迎用站内反馈表单或 Issue 提建议、报断链、荐 Skill。
