# 这个文件夹是什么 · 一页看明白

**指北 TrueNorth** —— 一个单文件 AI 内容导航站。把分散的 AI 文章、案例、工具、概念，
整理成互相关联的知识网络，带用户走完「**看懂 → 跑通**」这条路。

当前版本：**Skill-first MVP v1**（已冻结，无未解决 P0），现已 git 化并上线部署（含站内反馈）。

---

## 从哪看起

| 你想做什么 | 打开这个 |
| --- | --- |
| 直接用 / 体验网站 | 根目录的 `TrueNorth.html`（双击用浏览器打开，离线可用） |
| 看部署入口 | 根目录 `index.html`（自动跳转到 `TrueNorth.html`）+ `vercel.json` |
| 把项目发给外部 AI / 工程师做 review | 整个 `评审文档/` 文件夹 |
| 先快速了解项目全貌 | `评审文档/开发文档.md` |
| 看这一轮验收结论 | `评审文档/Review_Report.md` |
| 看待修问题清单 | `评审文档/P0_P1_P2_Issues.md` |

## 每个文件夹是什么

```
C:\Resume Now\TrueNorth\   （2026-07-23 由 c:\TrueNorth 迁入；git 仓库独立不变）
│
├─ TrueNorth.html        ← 网站本体。用浏览器打开它就是成品。
├─ index.html            ← 部署入口：自动跳转到 TrueNorth.html
├─ vercel.json           ← 部署配置
├─ 说明.md                ← 你正在看的这页。目录索引。
├─ README.md             ← 仓库说明（含内容基线）
│
├─ 评审文档/              ← 发给外部 AI / 评审用的，看这个文件夹就够。
│   ├─ 开发文档.md                    · 项目结构 / 数据模型 / 交互逻辑 / 流水线 / 已知限制
│   ├─ Review_Report.md              · 系统验收报告（v1 Freeze）
│   ├─ P0_P1_P2_Issues.md            · 问题清单（按 P0/P1/P2 分级）
│   ├─ Browser_Walkthrough_Report.md · jsdom 走查报告
│   └─ External_Review_Feedback.md   · 外部 reviewer 反馈
│
├─ 内容流水线/            ← 离线 Skill 提取/复核工具。后续批量扩容才用，前端不运行。
│   ├─ skill-extraction.md  · 流水线说明（命令在本目录下运行）
│   ├─ jobs.example.json    · 输入模板
│   ├─ scripts/             · extract-skills.py / merge-reviewed-skills.py
│   └─ data/                · 候选数据 json
│
├─ 论文终稿Review-SOP/     ← 独立交付物：论文终稿 review 可执行 SOP（HTML + MD）
├─ logos/                 ← 品牌 SVG 源文件（已 base64 内联进 HTML，保留作源）
│
└─（内部留存 · 不进对外仓库 / 分享包）工作记录/ · AI_.html · chat记录/ · *.zip
```

## 几条要记住的事

- **改网站只改 `TrueNorth.html` 一个文件**：单文件、内联 CSS/JS、无构建、无后端。
- **Skill-first**：Skill 是最小可复用能力单元；当前 Skill 库的 **45 条**为**人工精选入库**（全 reviewed），
  `内容流水线/` 的脚本是后续扩容才用的离线辅助，不是当前唯一入库方式。
- **改完必验证**：抽出两个 `<script>` 跑 `node --check`，0 error 才算过。
- **当前阶段**：MVP v1 已冻结、进入试用；以收敛、校准、小修为主，不堆功能、不拆单文件。
